# 配置管理指南

## 概述

oj problem import 使用 **Pydantic Settings** 进行环境变量管理,提供类型安全、自动验证和灵活的配置方式。

## 配置方式

### 1. .env 文件 (推荐)

在项目根目录创建 `.env` 文件:

```bash
cp .env.example .env
```

编辑 `.env` 文件:

```ini
LLM_OPENAI_API_KEY=sk-your-api-key-here
LLM_PARSER_MODEL=gpt-4
LLM_GENERATOR_MODEL=gpt-4
```

### 2. 系统环境变量

也可以直接设置系统环境变量:

```bash
# Linux/Mac
export LLM_OPENAI_API_KEY=sk-your-key

# Windows PowerShell
$env:LLM_OPENAI_API_KEY="sk-your-key"

# Windows CMD
set LLM_OPENAI_API_KEY=sk-your-key
```

### 3. 代码中动态配置

```python
from oj_engine.config import Settings

# 创建自定义配置
custom_settings = Settings(
    llm={
        "openai_api_key": "sk-your-key",
        "model": "gpt-4",
        "temperature": 0.2
    }
)
```

## 配置项说明

### LLM 配置

| 变量名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `LLM_OPENAI_API_KEY` | str | - | OpenAI API Key (必填) |
| `LLM_OPENAI_BASE_URL` | str | - | 自定义 API 端点 |
| `LLM_PARSER_MODEL` | str | gpt-4 | Parser 节点使用的模型 |
| `LLM_GENERATOR_MODEL` | str | gpt-4 | Generator 节点使用的模型 |
| `LLM_PARSER_TEMPERATURE` | float | 0.1 | Parser 温度参数 (0.0-1.0) |
| `LLM_GENERATOR_TEMPERATURE` | float | 0.2 | Generator 温度参数 (0.0-1.0) |

### Docker 配置

| 变量名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `DOCKER_DEFAULT_IMAGE` | str | python:3.10-slim | 默认 Docker 镜像 |
| `DOCKER_DEFAULT_MEM_LIMIT` | str | 512m | 内存限制 |
| `DOCKER_DEFAULT_CPU_QUOTA` | int | 50000 | CPU 配额 (微秒) |
| `DOCKER_DEFAULT_TIMEOUT` | int | 30 | 超时时间 (秒) |

### 工作流配置

| 变量名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `WORKFLOW_MAX_RETRIES` | int | 3 | 最大重试次数 |

## 使用示例

### 获取配置实例

```python
from oj_engine import settings, get_settings

# 使用全局单例
print(settings.llm.model)
print(settings.llm.temperature)

# 或使用函数获取
config = get_settings()
print(config.docker.default_image)
```

### 创建 LLM 客户端

```python
from oj_engine import settings

# 创建 Parser 用的 LLM 客户端
parser_llm = settings.get_llm_client(model_type="parser")

# 创建 Generator 用的 LLM 客户端
generator_llm = settings.get_llm_client(model_type="generator")
```

### 在节点中使用

节点已经自动使用配置化的 LLM:

```python
# parser.py 和 generator.py 中
from ..config import settings

llm = settings.get_llm_client(model_type="parser")  # 或 "generator"
```

## 配置优先级

配置的加载优先级 (从高到低):

1. **代码中显式传入** - 最高优先级
2. **环境变量** - 中等优先级
3. **.env 文件** - 较低优先级
4. **默认值** - 最低优先级

## 测试配置

运行配置测试脚本:

```bash
uv run python test_config.py
```

这将验证:
- ✓ 配置模块导入
- ✓ 配置值读取
- ✓ LLM 客户端创建
- ✓ .env 文件加载

## 常见问题

### Q: 如何验证配置是否正确加载?

```python
from oj_engine import settings

# 打印所有配置
print(f"Model: {settings.llm.model}")
print(f"Temperature: {settings.llm.temperature}")
print(f"API Key Set: {bool(settings.llm.openai_api_key)}")
```

### Q: 如何在开发环境和生产环境使用不同配置?

创建不同的 .env 文件:

```bash
# 开发环境
.env.dev

# 生产环境
.env.prod
```

在代码中指定:

```python
from pydantic_settings import SettingsConfigDict

class Settings(BaseSettings):
    model_config = SettingsConfigDict(
        env_file=".env.dev",  # 或 .env.prod
        env_file_encoding="utf-8"
    )
```

### Q: 支持哪些 LLM 提供商?

目前主要支持 OpenAI,但可以通过设置 `LLM_OPENAI_BASE_URL` 使用兼容 OpenAI API 的服务:

- OpenAI
- Azure OpenAI
- 本地部署的 LLM (如 Ollama, LM Studio)
- 其他兼容 OpenAI API 的服务

### Q: 如何调试配置问题?

启用详细日志:

```python
import logging
logging.basicConfig(level=logging.DEBUG)

from oj_engine import settings
```

## 最佳实践

1. **永远不要将 .env 文件提交到版本控制** - 已添加到 `.gitignore`
2. **使用 .env.example 作为模板** - 包含所有配置项但不含敏感信息
3. **为不同环境使用不同的配置文件** - dev, staging, prod
4. **定期轮换 API Key** - 提高安全性
5. **使用环境变量管理服务** - 如 HashiCorp Vault (生产环境)
